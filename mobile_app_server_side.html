Mobile Application (Server-Side)

My current company required a mobile application to control their gate motor operator. It must be able to send and receive messages to and from the gate motor opeator. Manage devices and view important information about the operator.

The company decided to out source the mobile application part due to resource constraints. Although the application was outsourced, I was involved in the layout and design of the application. I then got to develop the server-side of the application as well as the processing of the messages.

Multi-Tier

System Design

The system consists of the following: MySQL Database, Server-Side, Web API's and a message processor application

  • MySQL Database
  • Server-Side Implementation
  • Web API's
  • Message Processor Application
Front End
The front end is the user interface that the users interact with. The UI was designed using Html, CSS and Bootstrap that is an open source toolkit for controls like the buttons, cards, tables, badges, modal dialogs, etc. The web pages are located under the Views section of the MVC. Some JavaScript was used when passing data between the controller and the modal dialogs before it is displayed to the user. Razor which is Microsoft's client side scripting language was also used to do some client side processing.

When a client is created a profile and folder are created. The folder is used to store all the keypad files that are related to that client. An encryption format called AES (Advanced Encrytion Standard) is used to encrypt all the infomation in the database and folders.

User priviledges can also be set up. These changes can only be done by a master user. By setting these priviledges the user get access to only the functions that were enabled by the master user.
